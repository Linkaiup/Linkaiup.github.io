通俗地说,反射机制就是可以把一个类，类的成员(函数、属性)，当成一个对象来操作。也就是说，我们在运行程序的时候可以动态地去操作他们

​ 由 demo ，我们可以清晰地理解 JAVA 中反射 API 的使用方法。但是，这些 API 摆在那里，作为 JAVA 团体中的一名菜鸡，我好像只知道了反射有这么些优点：运行期类型判断优势、动态类加载、提高代码灵活度。但丝毫不知道如何使用这些优点，应该在哪里使用。因此我赶紧又学习了一番，认识了几个典型的反射应用场景。下面，我们就来看看在这些应用场景中，反射是如何发挥神力的。

JDBC 的数据库连接

​ 在JDBC 的操作中，如果要想进行数据库的连接，则必须按照以上的几步完成：

  1. 通过Class.forName()加载数据库的驱动程序 （通过反射加载，前提是引入相关了Jar包）
  2. 通过 DriverManager 类进行数据库的连接，连接的时候要输入数据库的连接地址、用户名、密码
  3. 通过Connection 接口接收连接
